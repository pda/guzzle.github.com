Class Inspector

Inspects configuration options versus defined parameters, adding default values where appropriate, performing type validation on config settings, and validating input vs output data.

The Reflection class can parse @guzzle specific parameters in a class's docblock comment and validate that a specified Guzzle\Common\Collection object has the default arguments specified, has the required arguments set, and that the passed arguments that match the typehints specified in the annotation.

The following is the format for @guzzle arguments:

Namespace: Guzzle\Service
Guzzle: argument_name [default="default value"] [required="true|false"] [type="registered constraint name"] [type_args=""] [doc="Description of argument"] Here's an example:
Guzzle: my_argument default="hello" required="true" doc="Set the argument to control the widget..."
